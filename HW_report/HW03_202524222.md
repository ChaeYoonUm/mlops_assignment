## 1. 데이터셋
- 데이터셋: Titanic.csv
- Task: 이진 분류 (Binary Classification)
- 타깃 변수: Survived

Titanic 데이터셋은 수치형 변수와 범주형 변수가 함께 포함된 대표적인 tabular 데이터셋이다. 따라서 Tree 계열 모델과 Boosting 계열 모델의 차이를 비교하기에 적합하다. 

## 2. 실험 구조
### 데이터 분할
Train: 534개 (60%)
Validation: 178개 (20%)
Test: 179개 (20%)

분류 task이기 때문에 클래스 비율이 크게 흔들리지 않도록 stratify를 사용했다.

### 사용 변수
입력 변수
- 수치형 변수: PClass, Age, SibSp, Parch, Fare
- 범주형 변수: Sex, Embarked

### 전처리
- 수치형 결측치: train 기준 median으로 대체
- 범주형 결측치: train 기준 mode로 대체

Tree 모델은 스케일링에 크게 민감하지 않기 때문에 스케일링은 적용하지 않았다.

### 평가 지표
- Accuracy
- F1-score
- ROC-AUC

Accuracy는 전체적인 예측 성능을 파악할 수 있는 평가 지표이다. F1-score는 precision과 recall을 반영하는 평가 지표이다. ROC-AUC는 threshold에 덜 민감하여 모델 비교에 적합한 평가지표이기 때문에 최종 모델 선택 기준은 validation ROC-AUC를 사용했다.

### Early stopping 결과 정리

Early Stopping을 적용하지 않았을 때는 train ROC-AUC가 0.9794였지만 validation ROC-AUC는 0.8630으로 오히려 낮아져서 과적합 결과를 보였다.
Early Stopping을 적용하자 train 성능은 0.8858로 낮아졌지만 validation ROC-AUC는 0.9037, test ROC-AUC는 0.8268로 개선되었다.
정리하면 Early Stopping은 validation과 test에서 더 나은 일반화 성능 높이는 역할을 했다. 따라서 해당 실험 결과는 Boosting 모델에서 validation 구조와 Early Stopping이 중요하다는 것을 보여준다.

## 6. 최종 모델 선택
최종 모델은 Validation ROC-AUC 기준으로 선택하였다. 그 결과 가장 높은 성능을 보인 모델은 다음과 같았다.
- 최종 선택 모델: Tuned XGBoost
    - learning_rate = 0.1
    - max_depth = 3
    - Validation ROC-AUC: 0.9082
    - Test ROC-AUC: 0.7989

validation ROC-AUC 성능 기준으로 모델을 선택했고 test 성능은 마지막 확인 단계에서만 사용하였다. 
이는 test 데이터가 모델 선택 과정에 개입하지 않도록 하기 위한 것으로 실험 실험 설계 측면에서 더 타당한 방식이라고 볼 수 있다.
흥미롭게도 test ROC-AUC만 보면 Early Stopping을 적용한 LightGBM이 0.8334로 더 높았다. 그러나 이번 과제에서는 validation ROC-AUC을 기준으로 최종 모델을 선택하고자 했기 때문에 최종 모델을 XGBoost로 선택했다. 
이는 단일 validation split만으로는 모델 선택이 항상 절대적이지 않을 수 있으며 더 안정적인 선택을 위해서는 향후 Stratified K-Fold cross validation도 고려할 수 있음을 보여준다.

## 7. 분석 코멘트

### 7-1. Random Forest와 Boosting의 차이를 결과에서 어떻게 느꼈는가?
Random Forest는 병렬적으로 여러 트리를 학습하여 variance를 줄이는 데 강한 모델이다. 실제로 baseline 성능도 안정적이었으며 validation 결과도 다른 모델과 큰 차이를 보이지 않았다. 
반면 Boosting 계열은 이전 모델의 오차를 다음 모델이 순차적으로 보완하는 방식이어서 train 성능을 더 향상시킬 수 있었다.
이번 실험 결과에서 Random Forest는 안정적인 baseline, Boosting은 성능 지향형 모델이라는 차이를 느낄 수 있었다.

### 7-2. XGBoost와 LightGBM 중 어떤 모델이 더 다루기 쉬웠는가?
이번 과제에서는 XGBoost가 조금 더 다루기 쉬웠다.
XGBoost는 learning_rate와 max_depth 중심으로 튜닝 방향을 직관적으로 설정할 수 있었으며 validation 기준으로 최종적인 최고 성능도 보였다.
반면 LightGBM은 baseline 성능은 강했지만 leaf-wise 성장 방식 때문에 num_leaves와 max_depth 제어에 더 민감했다. 
따라서 LightGBM은 빠르고 강력한 모델이지만 작은 데이터셋에서는 복잡도 제어에 주의를 기울여야 하는 모델임을 확인할 수 있었다.

### 7-3. Early Stopping은 어떤 효과가 있었는가?
Early Stopping을 적용하지 않으면 train 정확도는 매우 좋아졌지만 validation과 test 성능은 오히려 떨어졌다. 
반면 Early Stopping을 적용하면 train 정확도는 낮아졌지만 validation과 test 성능이 좋아졌다.
즉, Early Stopping은 과적합 방지, 일반화 성능 개선, 불필요한 학습 단계 감소라는 세 가지 측면에서 모두 효과적이었다.

### 7-4. 최종 모델을 왜 그렇게 선택했는가?
최종 모델은 test가 아니라 validation ROC-AUC를 기준으로 선택하였다. 이는 train 점수는 과적합으로 인해 쉽게 과대평가될 수 있고 test 데이터는 최종 일반화 성능 확인용으로 남겨 두어야 하기 때문이다. 그 결과 가장 높은 validation ROC-AUC를 보인 XGBoost tuned 모델을 최종 선택하였다.
따라서 이번 과제 실험을 통해 모델의 성능이 높을수록 train 정확도보다 validation 정확도를 기준으로 모델을 선택해야 한다는 점을 확인할 수 있었다.

## 8. 결론
이번 과제에서는 Titanic 데이터를 사용하여 Random Forest, XGBoost, LightGBM을 비교하였다. 
Random Forest는 안정적인 baseline 역할을 보였으며 Boosting 계열은 더 높은 성능을 낼 수 있었지만 validation 성능을 중심으로 모델을 평가하고 조정하는 것이 중요했다. 

튜닝 결과 XGBoost는 learning_rate=0.1, max_depth=3 설정에서 가장 높은 validation ROC-AUC를 보였다. LightGBM은 num_leaves를 포함한 leaf-wise 복잡도 제어가 중요함을 확인했다. 

또한 Early Stopping 실험을 통해 train 성능이 높다고 좋은 모델이 아닐 수 있으며 validation과 test에서의 일반화 성능을 기준으로 모델을 평가해야 한다는 점을 알 수 있었다.
